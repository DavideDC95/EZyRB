Online¶

Utilities for the online evaluation of the output of interest

Online Online phase
Online.run This method evaluates the new point value in the parametric space and returns the approximated solution.
Online.run_and_store This method evaluates the new point value in the parametric space and save the approximated solution on filename.
class Online(output_name, space_filename, dformat='cell')

Bases: object

Online phase

Parameters:
  • output_name (str) – the name of the output of interest.
  • space_filename (str) – the name of the file where the space has been saved.
  • dformat (str) – the data format to use for save the output to new file: if the parameter is “cell”, the approximated output will be saved to cell data, if the parameter is “point”, the approximated output will be saved to the point data. These are the only options available. Default is ‘cell’.
Variables:
  • output_name (str) – the name of the output of interest.
  • space_type (ParametricSpace) – the type of space used for the online phase.
  • dformat (str) – the data format to use for save the output to new file: if the parameter is “cell”, the approximated output will be saved to cell data, if the parameter is “point”, the approximated output will be saved to the point data. These are the only options available. Default is ‘cell’.
run(value)

This method evaluates the new point value in the parametric space and returns the approximated solution.

Parameters:value (array_like) – the point where the approximated solution has to be evaluated.
Returns:the approximated solution.
Return type:numpy.ndarray
run_and_store(value, filename, geometry_file=None)

This method evaluates the new point value in the parametric space and save the approximated solution on filename. It is possible to pass as optional argument the geometry_file that contains the topology on which the solution is projected.

Parameters:
  • value (array_like) – the point where the approximated solution has to be evaluated.
  • filename (str) – the file where the approximated solution is projected.
  • geometry_filename (str) – the file that contains the topology to use for the solution projection.
