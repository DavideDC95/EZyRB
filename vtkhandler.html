Vtk format file handler class. You are NOT supposed to call directly this class constructor (use FileHandler constructor instead)

Parameters:

filename (str) – the name of the file to handle.
Variables:
  • _filename (str) – name of file to handle
  • _cached_data (vtkPolyData) – private attribute to store the last polydata processed
_read_polydata()

This private method reads the given filename and return a vtkPolyData object containing all informations about file; to avoid useless IO operation on the same file, it stores polydata of the last file parsed and if user ask for this file, the polydata previously stored is returned.

Returns:polydata containing information about file.
Return type:vtkPolyData
_save_polydata(data, write_bin=False)

This private method saves into filename the data. data is a vtkPolydata. It is possible to specify format for filename: if write_bin is True, file is written in binary format, otherwise in ASCII format. This method save cached polydata to reduce number of IO operations.

Parameters:
  • data (vtkPolyData) – polydatat to save.
  • write_bin (bool) – for binary format file.
get_all_output_names()

Return the list of all the output names by point and the list of all the output names by cell.

Returns:point output name, cell output name
Return type:tuple(list(str), list(str))
get_dataset(output_name, datatype='point')

This method reads the given filename and returns a numpy array containing output_name field for each point (if datatype is ‘point’) or for each cell (if datatype is ‘cell’). If output_name is not a field in file, it raises exception.

Parameters:
  • output_name (str) – the name of the output of interest to extract from file.
  • datatype (str) – a string to specify if point data or cell data should be returned. Default value is point.
Returns:

the matrix n_components-by-n_elements containing the extracted output.
Return type:

numpy.ndarray
get_geometry(get_cells=False)

This method reads the given filename and returns points and cells of file. If get_cells is True, it computes the list that contain index of points defining cells, otherwise the list is not computed and set to None (less expensive).

Parameters:get_cells (bool) – flag to compute cells list or not. Default is false.
Returns:the n_points-by-3 matrix containing the coordinates of the points, the n_cells list containing, for each cell, the iindex of the points that define the cell (if computed).
Return type:numpy.ndarray, list(numpy.ndarray)
set_dataset(output_values, output_name, datatype='point', write_bin=False)

Writes to filename the given output. output_values is a matrix that contains the new values of output to write, output_name is a string that indicates name of output to write.

Parameters:
  • output_values (numpy.ndarray) – the n_points-by-n_components matrix containing the output values.
  • output_name (str) – name of the output.
  • datatype (str) – a string to specify if point data or cell data should be returned. Default value is ‘point’.
  • write_bin (bool) – flag to write in the binary format. Default is false.
set_geometry(points, cells, write_bin=False)

This method writes to filename a new data defined by points and cells.

Parameters:
  • points (numpy.ndarray) – matrix n_points-by-3 containing coordinates of all points.
  • cell (list(array_like)) – list that contains for each cell the list of the indices of points that define the cell.
  • write_bin (bool) – flag to write in the binary format. Default is false.
